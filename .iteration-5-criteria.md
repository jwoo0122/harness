# Iteration 5 Criteria — Extension Refactor Slice 1

## Goal
Start the `extensions/index.ts` decomposition using the agreed sequence:

1. Plan
2. Validate (prepare)
3. Implement
4. Validate

This iteration covers only the **prompt-loading seam**. It intentionally does **not** extract registry logic, bash policy logic, web tooling, subagent result formatting, or lifecycle orchestration.

## Acceptance Criteria

### AC-5.1 — Validation prepare entrypoints exist
Before any refactor extraction, the repo must provide repeatable validation commands in `package.json`:
- `validate:extensions:prepare`
- `validate:extensions`

### AC-5.2 — Prepare gate works without peer dependency installs
`npm run validate:extensions:prepare` must pass on this repo **without** requiring local installs of:
- `@mariozechner/pi-coding-agent`
- `@mariozechner/pi-tui`
- `@sinclair/typebox`

The prepare gate must verify:
- the target seam still exists in `extensions/index.ts`
- adjacent non-scope helpers are still present
- all flat `agents/*.md` prompt files exist and are non-empty
- local Node can import a self-contained `.ts` module with `--experimental-strip-types`

### AC-5.3 — Prompt seam is extracted into a leaf module
Create `extensions/agent-prompts.ts` and move exactly these five functions out of `extensions/index.ts`:
- `loadAgentPrompt`
- `buildExploreSubagentSystemPrompt`
- `buildExploreSubagentTask`
- `buildExecuteRoleSystemPrompt`
- `buildExecuteRoleTask`

### AC-5.4 — Extracted module is truly leaf-like
`extensions/agent-prompts.ts` must:
- use only Node built-ins
- own its own prompt cache / package-root resolution
- not import from `extensions/index.ts`

### AC-5.5 — Root wiring is updated without expanding scope
`extensions/index.ts` must import the extracted prompt helpers from `./agent-prompts.js` and must no longer define the five moved functions.

### AC-5.6 — Post-implementation validator proves contract parity
`npm run validate:extensions` must pass and verify:
- prompt loading returns normalized file contents for all `agents/*.md`
- repeated prompt loads are stable
- missing prompt path errors include the resolved absolute path
- representative explore/execute system-prompt builders return exact expected strings
- representative explore/execute task builders return exact expected strings
- source-shape checks confirm the extraction is real

### AC-5.7 — Syntax gate remains green
The post-implementation validation command must also include syntax checks for the touched extension files, at minimum:
- `extensions/index.ts`
- `extensions/agent-prompts.ts`
- `extensions/subagents.ts`

## Explicit Non-Goals
- no extraction of verification-registry logic yet
- no extraction of bash-policy logic yet
- no refactor of `extensions/subagents.ts`
- no refactor of `extensions/compact-tool-renderers.ts`
- no tool name or tool schema changes
- no `agents/` layout changes
