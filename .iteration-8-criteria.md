# Iteration 8 Criteria — Explore Gate + Live UI Cleanup

## Goal
Finish the remaining explore-mode improvements so that:

1. live subagent progress stays entirely inside tool-call rows
2. external-evidence enforcement happens before an explore run completes
3. explore child subagents can be steered to keep researching and then conclude once evidence is sufficient
4. validation is deterministic for VER

This iteration does **not** redesign the `/explore` debate protocol, the generic subagent runtime, or the verification registry tools.

## Acceptance Criteria

### AC-8.1 — No separate subagent widget/banner surface remains
`extensions/index.ts` must not maintain or render a dedicated live subagent widget/banner state.

Expected shape:
- `updateUI()` clears the harness widget surface instead of populating it
- dead live-widget-only batch state/helpers are removed or simplified away
- footer status can remain, but live subagent detail must not depend on a separate widget

### AC-8.2 — Collapsed subagent tool rows remain per-subagent and activity-rich
Collapsed subagent tool rendering must continue to produce:
- one summary/header line for the batch
- one row per subagent/persona/role underneath
- activity-rich content that prefers live tool activity, then last tool activity, then assistant preview/output

This must remain true for:
- explore subagents
- execute subagents
- generic `harness_subagents`

### AC-8.3 — Parent explore evidence gate is enforced pre-completion
The main `/explore` evidence gate must move off the post-completion `agent_end` retry path and be enforced before completion.

Expected behavior:
- explore gating is evaluated from `turn_end`
- failed pre-completion gate results are sent with `deliverAs: "steer"`
- the agent keeps exploring/revising until the evidence gate passes
- the old `followUp` retry path is not the primary explore gate anymore

Parent completion requirements must still include:
- isolated OPT/PRA/SKP/EMP subagent evidence
- external discovery/search activity
- source inspection/fetch activity
- explicit external URL citations in the answer

### AC-8.4 — Explore child subagents get child-safe gating and prompt handling
Explore child subprocesses must consume `HARNESS_SUBAGENT_MODE` and behave differently from the parent explore orchestrator.

Expected behavior:
- child explore startup/prompt handling must not reuse the parent explore prompt that instructs recursive `harness_subagents` use
- child explore subagents can be steered to keep researching when evidence is insufficient
- once child evidence is sufficient, the harness can steer them that they are ready to conclude
- child completion still requires explicit URL citations in the returned answer

### AC-8.5 — Validation covers the new gate deterministically
`package.json` `validate:extensions` must cover any new helper module/test added for this work.

Validation must make it straightforward for VER to confirm:
- the explore gate helper behavior for parent vs child thresholds
- `turn_end` + `steer` wiring is present
- child explore prompt handling exists
- dead live-widget-only helpers are gone
- collapsed per-subagent rows still remain in source

## Explicit Non-Goals
- no redesign of `/explore` skill markdown output format
- no recursive child subagent orchestration
- no verification registry schema changes
- no execute-mode role/protocol redesign
- no change to tool names or tool schemas beyond what this increment needs
