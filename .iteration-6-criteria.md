# Iteration 6 Criteria — Extension Refactor Slice 2

## Goal
Continue the `extensions/index.ts` decomposition using the agreed sequence:

1. Plan
2. Validate (prepare)
3. Implement
4. Validate

This iteration covers only the **verification registry storage seam**. It intentionally does **not** extract the `harness_verify_register` / `harness_verify_list` tool handlers themselves, bash-policy logic, web tooling, subagent result formatting, or lifecycle orchestration.

## Acceptance Criteria

### AC-6.1 — Prepare entrypoint is repointed to the registry seam
Before registry extraction, `package.json` must wire:
- `validate:extensions:prepare` → registry-specific prepare validation

The stale prompt-seam prepare hook must no longer be the active prepare entrypoint.

### AC-6.2 — Registry prepare gate works without peer dependency installs
`npm run validate:extensions:prepare` must pass on this repo without requiring local installs of:
- `@mariozechner/pi-coding-agent`
- `@mariozechner/pi-tui`
- `@sinclair/typebox`

The prepare gate must verify by text/FS inspection that:
- the registry seam still exists in `extensions/index.ts`
- the verify tool handlers still exist in `extensions/index.ts`
- the current handlers still call local `readRegistry` / `writeRegistry`
- extraction has not already happened yet

### AC-6.3 — Registry storage seam is extracted into a leaf module
Create `extensions/verification-registry.ts` and move the registry storage seam out of `extensions/index.ts`, including:
- `VerificationEntry`
- `VerificationRegistry`
- registry path constants / ownership
- `readRegistry`
- `writeRegistry`

### AC-6.4 — Extracted registry module is truly leaf-like
`extensions/verification-registry.ts` must:
- use only Node built-ins
- not import from `extensions/index.ts`
- not import `@mariozechner/*` or `@sinclair/typebox`

### AC-6.5 — Root wiring is updated without expanding scope
`extensions/index.ts` must import registry helpers from `./verification-registry.js` and must no longer define the extracted registry seam.

The following must remain in `extensions/index.ts` for this increment:
- `harness_verify_register`
- `harness_verify_list`

### AC-6.6 — Post-implementation validator proves registry behavior parity
`npm run validate:extensions` must pass and verify:
- the existing prompt validation still runs
- the registry module can be syntax-checked
- missing registry file returns the same empty/default registry shape
- `writeRegistry` creates `.harness/verification-registry.json`
- read-after-write round-trips representative data correctly
- malformed JSON falls back to the same empty/default registry shape
- written JSON remains pretty-printed and ends with a trailing newline
- source-shape checks confirm the extraction is real

### AC-6.7 — Scope does not expand into verify tool extraction
No changes in this iteration may extract or redesign:
- `harness_verify_register`
- `harness_verify_list`
- registry summary/output text
- registry schema string
- registry file path

## Explicit Non-Goals
- no extraction of verify tool registration/handlers yet
- no extraction of bash-policy logic yet
- no refactor of `extensions/subagents.ts`
- no refactor of `extensions/compact-tool-renderers.ts`
- no tool name or tool schema changes
- no `agents/` layout changes
